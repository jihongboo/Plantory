# PlantInformation CloudKit Seed v1

This folder contains the first public PlantInformation catalog for Plantory.

Upload target:
- CloudKit database: Public Database
- Record type: `PlantInformation`
- Asset field: `image`
- Source data: `catalog.json`
- Image files: `images/*.png`
- cktool files: `cktool-records/`

The app should treat CloudKit Public Database as the source of truth for the plant encyclopedia. The app caches `PlantInformation` records in SwiftData for local-first display and refreshes them from Public Database.

## Record Identity

Use `catalogID` as the stable product identifier. Recommended CloudKit `recordName` values match `catalogID`.

Example:

```text
catalogID: monstera-deliciosa
recordName: monstera-deliciosa
```

## Image Guidance

Each `catalogID` maps to a transparent-background pixel plant PNG in `images/` using `<catalogID>.png`. Upload that PNG to the CloudKit `image` Asset field for the matching record.

Images are generated by:

```bash
python3 CloudKitSeed/PlantInformation/v1/generate_pixel_images.py
```

## cktool Upload

The seed package includes a helper that converts `catalog.json` into cktool field files:

```bash
python3 CloudKitSeed/PlantInformation/v1/prepare_cktool_records.py
```

Before the first record upload, create the `PlantInformation` record type in the CloudKit Console or import a schema with a management token. If the record type is missing, `cktool create-record` returns `not-found`.

Before uploading records, save a CloudKit user token:

```bash
xcrun cktool save-token --type user --method keychain
```

Schema management commands require a separate management token:

```bash
xcrun cktool save-token --type management --method keychain
```

Then upload to the Public Database:

```bash
export CK_TEAM_ID="CR7BE3B56R"
export CK_CONTAINER_ID="iCloud.com.hongbo.Plantory"
export CK_ENVIRONMENT="development"

CloudKitSeed/PlantInformation/v1/cktool-records/upload_all.sh
```

Notes:

- `xcrun cktool` is available from Xcode even if `cktool` is not directly in `PATH`.
- This seed uses `catalogID` as the stable business identifier.
- The local Xcode `cktool create-record` help does not expose a custom record name option, so repeated uploads should first query or delete existing records by `catalogID` to avoid duplicates.
